| Boundary | Description |
|----------|-------------|
| service → Redis | Pending task JSON (user-derived content) persisted under a guessable-but-random pendingId |
| Redis → service | Pending read back and later sent to ClickUp |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Elevation of Privilege | claimPending | mitigate | GETDEL makes the claim atomic so two concurrent confirms cannot both create (idempotency primitive) |
| T-03-05 | Information Disclosure | pending TTL | mitigate | pending:<id> carries a 1h TTL so abandoned previews self-expire; pendingId is a random uuid |
| T-03-06 | Tampering | preview display values | mitigate | Cliente/asignados rendered from inverted config maps, not raw LLM strings — human confirms what will actually be sent (Pitfall 4/UX) |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages (reuses @upstash/redis + luxon already in deps) |
</threat_model>
