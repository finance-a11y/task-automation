## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Slack → /api/slack/interactions | Untrusted POST; button clicks; signature must be verified over raw body |
| service → ClickUp | Outbound task creation triggered by a confirmed click |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing | interactions endpoint | mitigate | @vercel/slack-bolt VercelReceiver verifies the Slack signing-secret HMAC over the raw body (same discipline as events); no hand-rolled HMAC |
| T-03-08 | Tampering | duplicate confirm create | mitigate | claimPending (GETDEL) guarantees exactly-one createTask even on double-click / Slack redelivery (Pitfall 1/8) |
| T-03-09 | Denial of Service | inline create before ack | mitigate | ack() called first; createTask runs in adapter waitUntil — stays under Slack's 3s deadline |
| T-03-10 | Repudiation | lost confirmation on cold start | mitigate | Pending lives in Redis keyed by pendingId carried in the button value; confirm reloads it, never re-parses (Pitfall 8) |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages (reuses existing deps + node:crypto + global fetch) |
</threat_model>
