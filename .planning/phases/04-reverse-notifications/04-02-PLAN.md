<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webhook payload → Slack post | Attacker-controllable fields (task_id, status labels) flow into a Slack message |
| webhook → task2thread | Only task_ids the bot itself stored may trigger a post |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Spoofing/Elevation | task scoping | mitigate | getThreadForTask gate: only tasks in task2thread (bot-created) notify; unknown task_id silently dropped |
| T-04-05 | Tampering | webhook redelivery | mitigate | markWebhookDeliveryOnce dedup on (event+task_id+history_item id) |
| T-04-06 | Information Disclosure | getTask errors | mitigate | error text carries status + body but never the Authorization token (mirrors createTask) |
| T-04-07 | DoS/Echo | feedback loop | accept | Flow B only posts (never re-ingests its own posts); scoped to bot-created tasks — loop not reachable |
| T-04-SC | Tampering | npm installs | accept | No new external packages introduced in this plan |
</threat_model>
