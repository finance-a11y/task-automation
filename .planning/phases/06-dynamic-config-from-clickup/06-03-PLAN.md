| Slack users.info → bot | reads teammate email (PII) to map identities |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Elevation / Spoofing | refresh-config endpoint | mitigate | Timing-safe secret gate reusing the diag pattern (SLACK_SIGNING_SECRET); 401 on mismatch. Phase 8 hardens diag + this endpoint further |
| T-06-07 | Info Disclosure | teammate emails (PII) via users.info | mitigate | Emails used only in-memory for matching + cached under cfg:slackmap as the resolved member ID (not the raw email); never logged or returned in responses |
| T-06-08 | DoS | refresh endpoint cache-clear spam | accept | Only clears short-TTL cache (forces a re-fetch, bounded by ClickUp retry caps); gated by secret. Rate-limiting deferred to Phase 8 |
| T-06-09 | Availability | provider/email failure breaking capture | mitigate | All provider + email resolution wrapped to degrade to static maps; capture/preview flow never blocked (DYN-05) |
| T-06-SC | Tampering | npm installs | accept | No new packages — WebClient (@slack/web-api) and crypto already in use |
