<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet → /api/slack/events | Untrusted POSTs claiming to be Slack cross here |
| bot output → Slack channel | The bot's own posts could be re-ingested (echo) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | /api/slack/events | mitigate | Bolt/@vercel/slack-bolt verify Slack HMAC over the raw body (INGEST-01); reject invalid signatures |
| T-01-02 | Tampering | request body | mitigate | Signature computed over exact raw bytes via the adapter — no JSON re-serialization before verify (Pitfall 2) |
| T-01-03 | Replay / DoS | retried/replayed events | mitigate | Reject timestamps >5min old (replay window) + markEventOnce dedup on event_id (INGEST-03, Pitfall 1) |
| T-01-05 | Elevation / loop | echo of own/bot messages | mitigate | isProcessableMessage filters bot_id / own user / non-root / wrong channel (INGEST-04, Pitfall 3) |
| T-01-04 | Information disclosure | tokens in logs | mitigate | Tokens only from env; processor logs failures without secrets |
</threat_model>
